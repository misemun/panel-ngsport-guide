# panel-ngsport-guide

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的仓库。页面内容以静态 HTML 文件形式存放，可直接通过 GitHub Pages 等静态托管服务访问。本仓库不针对任何特定域名或网站，仅作为通用型页面存档与分发用途。

## 目录结构

```
panel-ngsport-guide/
├── index.html          # 首页 / 导航页（可选）
├── pages/              # 存放各独立 HTML 页面文件
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 页面引用的静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/` 目录为主要页面存放位置，每个 HTML 文件为一个独立页面。
- `assets/` 目录存放页面所需的样式、脚本与图片资源，可按需组织子目录。
- 根目录下的 `index.html` 可作为入口页或导航页，具体内容视归档需求而定。

## 页面归档说明

- 所有页面均为静态 HTML，不依赖后端服务。
- 页面之间彼此独立，无强制关联，可单独部署或访问。
- 页面文件名建议使用英文小写字母、数字与短横线，避免特殊字符。
- 若页面需要引用外部资源（如字体库、CDN 库），请尽量使用通用 CDN 链接，或自行托管至 `assets/` 目录。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 按需修改 `pages/` 中的 HTML 文件，或新增页面。
3. 通过本地浏览器直接打开 HTML 文件即可预览。
4. 如需在线访问，可启用 GitHub Pages（Settings → Pages → 选择 main 分支及 `/` 目录），稍等片刻后即可通过 `https://<你的用户名>.github.io/panel-ngsport-guide/` 访问。
